# Proposal: Switch To Official OpenSpec

## Why

This repository currently carries a repo-local localized OpenSpec workflow variant in the committed Codex skills. Official OpenSpec now provides mature multi-language support through `openspec/config.yaml`, so keeping a forked CLI naming convention adds maintenance cost without enough benefit.

## What Changes

- Replace repo-tracked Codex OpenSpec skills with the latest official `core` skills generated by `@fission-ai/openspec`.
- Preserve the existing Chinese project context and artifact-writing rules in `openspec/config.yaml`.
- Document the official OpenSpec install and update workflow for contributors.

## Non-Goals

- No change to the published `sync-ai-context` runtime API or `context-sync` CLI behavior.
- No rewrite of product-facing README content.
- No compatibility shim that keeps the legacy localized CLI name available going forward.

## Impact

- Contributors use the official `openspec` CLI instead of the repo-local localized conventions.
- Repo-tracked Codex skills become deterministic snapshots of official upstream output.
- Existing `openspec/specs/` content and archived changes remain unchanged.
