## 为什么

当前团队在多 AI 工具协作时，规则与技能文件容易分叉，且在本地开发与云端/CI 场景中对 symlink 的支持不一致，导致上下文同步成本高且不稳定。现在需要一个开源优先、可配置的同步工具，在保持 single-source 的同时支持本地与云端的不同落地策略。

## 变更内容

- 新增一个通用上下文策略同步工具（基于 rslib 构建的 npm 包 + CLI）。
- 支持以独立目录或某个工具目录作为 single-source（例如直接使用 `.claude/skills` 作为源）。
- 支持多目标同步与两种核心落地模式：`link` 与 `copy`。
- 支持 `check` 能力：
  - `link` 目标检查符号链接是否指向预期源路径。
  - `copy` 目标检查文件哈希一致性，并报告 modified/missing/extra。
- 支持 `dry-run` 与 `force`，用于安全预览与冲突处理。
- 支持声明式配置文件驱动（source、targets、mode、policy），以便后续扩展更多工具适配器。
- 采用 TDD 开发模式，要求先编写测试用例，再实现功能代码。
- 测试框架统一使用 `rstest`，覆盖单元测试与命令级端到端测试。
- 一期明确不包含云端模板拉取与远程 registry。

## 功能 (Capabilities)

### 新增功能
- `context-strategy-sync`: 提供 single-source 驱动的 rules + skills 多工具同步、校验和冲突治理能力。

### 修改功能
- 无

## 影响

- 新增 CLI 命令与同步引擎模块。
- 新增配置文件 schema 与配置解析逻辑。
- 新增文件系统操作与哈希校验逻辑。
- 新增测试夹具与端到端命令测试。
- 测试体系强制使用 `rstest` 并以测试先行保障行为正确性。
- 对外 API 和 CLI 语义形成后续扩展的稳定契约。
