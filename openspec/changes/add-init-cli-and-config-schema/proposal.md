## 为什么

当前配置文件需要用户手写，首次接入成本高且容易出现字段错误；当用户在编辑器中修改 JSON 时，也缺少统一的 schema 提示与校验来源。现在需要提供一个交互式初始化入口和可公开引用的 JSON Schema，降低上手门槛并减少配置错误。

## 变更内容

- 新增 `init` 命令，通过交互式 CLI 引导用户初始化配置文件。
- 在 `init` 流程中提供常用工具预设选择（可多选），并一键生成对应 `targets` 与基础策略配置。
- `init` 支持选择输出路径与覆盖确认，避免误覆盖已有配置。
- 增加官方 JSON Schema（与配置结构保持同步），用于编辑器智能提示与静态校验。
- 约定并文档化 GitHub 可访问的 schema 发布地址，允许用户在配置中通过 `$schema` 引用。
- 保持现有配置加载与执行兼容：旧配置可继续工作，不强制要求立刻迁移。
- 非目标：本次不引入远程模板市场、云端配置拉取或在线 registry。

## 功能 (Capabilities)

### 新增功能
- 无

### 修改功能
- `context-strategy-sync`: 扩展配置体验，新增交互式 `init` 入口与可公开引用的 JSON Schema 发布/引用约定。

## 影响

- CLI 命令层：新增 `init` 子命令及交互流程。
- 配置模型层：补充 schema 文件、`$schema` 引用约定与版本兼容策略。
- 文档与发布流程：新增 schema 地址说明与维护规则。
- 测试：新增 `init` 交互流程、预设生成结果、schema 有效性与兼容性测试。
