# Security Policy

## Supported Versions

Only the latest published minor version is actively supported for security fixes.

## Reporting a Vulnerability

Please report vulnerabilities privately:

- Open a private security advisory in GitHub, or
- Email the maintainers via the contact listed in repository settings.

Include:

- Affected version
- Reproduction steps
- Impact summary
- Suggested mitigation (if available)

## Disclosure Process

1. Maintainers acknowledge report receipt.
2. Issue is triaged and a fix plan is prepared.
3. Fix is released through CI release workflow.
4. Public disclosure is published with remediation guidance.
